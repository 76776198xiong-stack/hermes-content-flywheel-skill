# Content Flywheel Workflow

## 1. Onboarding

Use onboarding only when the creator has no profile yet or asks to reset direction.

Minimum profile:

```markdown
# Creator Profile

## Creator
- name:
- public identity:
- strongest lived experience:
- topics with unfair advantage:
- topics to avoid:

## Audience
- audience:
- pain:
- desired result:
- what they already tried:
- what they misunderstand:

## Content Lanes
| lane | purpose | examples | CTA |
|---|---|---|---|

## Voice
- natural phrases:
- forbidden phrases:
- rhythm:
- examples that sound like the user:

## Products / Offers
| offer | who it helps | problem | current status | CTA |
|---|---|---|---|---|
```

Do not make onboarding a blocker. If the user gives a thought before the profile is complete, produce the content and infer missing profile items as assumptions.

## 2. Daily One-Fish

Input can be:

- a voice transcript
- a messy note
- a conversation excerpt
- a user question
- a published draft
- a topic list

Steps:

1. Extract 1-3 possible "fish."
2. Score each fish quickly:

| Score | Meaning |
|---|---|
| 0 | no signal |
| 1 | weak / needs more context |
| 2 | usable now |

Criteria: real pain, distinct view, platform hook, business relevance.

3. Choose one fish for today.
4. Produce a content package.
5. Ask for one correction that would make it more true.

## 3. Short Video Script Shape

Use for Douyin, Reels, Shorts, Xiaohongshu video, or video account.

```markdown
**开头钩子**
One line. Concrete, spoken, not a thesis title.

**正文**
1. Common belief / common behavior.
2. Why it fails or misses the key variable.
3. The creator's deeper view.
4. A concrete scene or example.
5. A minimal action, distinction, or decision.

**收束**
One memorable sentence + light CTA.
```

Default length:

- 30-60 seconds for quick thought
- 60-120 seconds for one insight with example
- 3-5 minutes only when user asks for a deep script

## 4. One-Fish-Many-Uses

Same idea, different platform jobs:

| Format | Job | Output |
|---|---|---|
| Short video | attention + trust | spoken script + cover/title |
| X/Twitter | idea spread | 1 short post or thread |
| Xiaohongshu text | save/share | title + structured note |
| WeChat article | depth | outline + intro + sections |
| Community answer | intimacy | direct answer with context |
| Course/product note | asset | principle + steps + examples |

Do not preserve the same wording across all platforms. Preserve the core insight and adapt the job.

## 5. Product-Aware Content

When the creator has a product or community, content should not hard-sell by default. Use one of three CTA levels:

| Level | Use when | CTA |
|---|---|---|
| Soft | trust-building content | "我后面会继续拆这个" |
| Medium | content directly relates to a product problem | "如果你也卡在这里，可以来社群继续问" |
| Direct | user asks for conversion copy | clear offer + who it is for + next step |

Never promise a product result that does not exist yet. If the product is unfinished, say the current usable method and that the system is being built in public.

## 6. Review Loop

When the user provides data, ask for or infer:

- platform
- date/time posted
- views
- watch completion or read completion if available
- likes/comments/saves/shares
- follows/leads/sales if available
- representative comments
- user's own feeling

Review output:

```markdown
**结论**
good / mixed / weak / inconclusive

**可能原因**
content, hook, topic, timing, packaging, audience mismatch, production quality

**可复用经验**
rules to keep

**下次实验**
one variable to test next

**回流**
new topic ideas, style rules, product questions
```
